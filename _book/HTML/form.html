第二节

form表单笔记

1、form标签的使用

  1. name属性  用来区分多个表单
  2. action属性 用来指明处理表单数据的人是谁
  3. method 属性 表示数据的传输方式
    • post  数据量较大 安全性高
    • get  数据量较少 安全性低
姓名： 
<form  name="reg" action="reg.php" method="post">
            <input type="text" name="username" id="" value="" /><br />

            <input type="submit" value="确定"/>
</form>

备注:form表单是用户用来提交数据的方式之一

参考链接

2、input标签的使用

表单家族里面最大的单个元素了，因为他有多个类型 所以功能也是最多的

  1. type属性  表示输入框的具体类型
    • text 文字输入框  用于输入文字
    • password  密码输入 输入的文字会以*表示
    • ridio 单选框   name属性要一致的，以表示同一组选项
    • checkbox 复选框 ,  name属性最好一致的，还要加上[]，以表示同一组选项 checked 表示默认选中
    • search 搜索框  用于搜索
    • bottom 普通按钮 需要脚本语言配合
    • submit 提交按钮 用来确定数据的提交
    • reset 重置按钮 用来清除表单的输入数据
    • image 图片按钮 点击图片时触发按钮功能
    • hidden  隐藏某些特殊的数据
    • file  文件上传，但是必须要form表单的enctype属性设置为multpart/form-data,method属性必须设置为post,如果需要多文件上传，可以增加一个multiple属性
  2. name属性  用来区分的多个input元素，并且该属性是必须的。
  3. value属性  用来装载具体的值，可以是用户输入的，也可以是预先设置好的
  4. maxlengh 表示输入数据的最大字数，不分中英文
  5. readonly  是一个单属性，通常只需要把属性名字写上即可，表示该属性只读，不允许修改，不影响数据的提交
  6. disebled  也是一个单属性，表示禁用，不允许修改和提交
  7. placeholder  用来显示提示
<input type="text" name="username"value="bgg"
    maxlength="15" placeholder="请输入用户名" readonly disabled/>
<input type="password" name="pw"/><br />

备注:要注意文本域的值是写在标签对之间，并且把所有的空格和换行记录并显示出来,如果想要禁止改变文本域的大小需要css属性的resize: none;
